# Machine Learning Layer

## Purpose

The ML layer is responsible for incorporating predictive workloads into the PT Frozen Foods Lakehouse platform.

This layer extends the existing analytical architecture and enables:

- feature engineering
- model training
- experiment tracking
- predictive analytics
- batch inference
- future MLOps evolution

The ML workflows are built on top of governed datasets already available in the platform.

---

# Exploratory Workloads

Exploratory notebooks are intended for:

- EDA
- feature analysis
- experimentation
- model comparison
- hypothesis validation

Characteristics:

- notebook-based
- visualization-oriented
- iterative development
- experimentation-friendly

These notebooks are not considered production artifacts.

---

# Processing Workloads

Processing notebooks are intended for:

- reusable feature engineering
- Delta table generation
- deterministic transformations
- scheduled execution

Characteristics:

- `.py` notebooks
- minimalistic structure
- production-oriented execution
- optimized for performance and cost

These notebooks follow the same standards already adopted across Bronze, Silver, Gold and Analytics layers.

# Spark vs Pandas Strategy

The platform adopts a hybrid processing strategy.

## PySpark

Used for:

- scalable transformations
- distributed aggregations
- feature engineering
- Delta materialization

---

## Pandas

Used for:

- exploratory analysis
- local model experimentation
- smaller ML datasets
- evaluation workflows

This approach preserves scalability while avoiding unnecessary distributed complexity during experimentation.
